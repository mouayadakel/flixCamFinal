## 2) المعلومات المطلوبة لموقع عام كامل

### 2.1 أعمال وهوية

- **الجمهور المستهدف** (من هم؟ إنتاج، وكالات، فريلانسرز، محتوى).
- **السوق/الموقع** (سعودية فقط؟ مدن متعددة؟ لغة: عربي فقط، إنجليزي فقط، أو الاثنين مع لغة افتراضية).
- **التموضع** (جملة واحدة + 2–3 مميزات تنافسية).
- **النبرة** (احترافي فقط أم مزيج احترافي + إبداعي).

### 2.2 هيكل الموقع (صفحات وأقسام)

- **الصفحات الأساسية:** رئيسية، عن، اتصل بنا، معدات، استوديوهات، تسعير، أسئلة شائعة، تسجيل دخول/تسجيل. أي صفحات إضافية (مدونة، وظائف، شركاء).
- **الصفحة الرئيسية:** الأقسام المطلوبة (هيرو، معدات مميزة، استوديوهات، كيف نعمل، شهادات، CTA، فوتر).
- **المعدات/الاستوديوهات:** كتالوج عام فقط، أو "اطلب عرض سعر" أو "سجّل دخول واحجز". فلاتر (فئة، ماركة، سعر، توفر).
- **الفوتر:** الروابط المطلوبة (عن، اتصل، أسئلة شائعة، شروط، خصوصية، سوشال).

### 2.3 المحتوى

- **نصوص:** جاهزة منكم أم نستخدم placeholder؟
- **وسائط:** شعار، صور هيرو، صور معدات/استوديوهات – جاهزة أم placeholder؟
- **قانوني:** نصوص الشروط وسياسة الخصوصية (سعودي) جاهزة أم نضع هيكل فقط؟

### 2.4 التصميم

- **مواقع مرجعية** (1–3) لأسلوب أو تخطيط تحبونه.
- **ألوان العلامة** (هيكس) أو "اقترح palette".
- **RTL:** كامل عربي أولاً أم LTR للإنجليزي و RTL للعربي فقط؟
- **موبايل:** أولوية موبايل أولاً مع أي متطلبات خاصة.

### 2.5 الوظائف والتدفقات

- **الحجز/العرض:** "اطلب عرض سعر" فقط أم "تحقق من التوفر" أيضاً؟ بعد طلب عرض: تخزين في DB و/أو إرسال إيميل؟
- **تسجيل الدخول:** أين يوجّه (مثلاً `/login`) وبعد الدخول إلى `/portal`.
- **تفاصيل المعدة/الاستوديو:** ما يظهر بدون تسجيل (اسم، صورة، وصف قصير، نطاق سعر، زر "اطلب عرض" أو "احجز"). أي شيء يُخفى حتى تسجيل الدخول؟
- **النشرة/جمع leads:** هل مطلوب "اشتراك" (إيميل) في الفوتر أو popup، وإلى أين تُرسل البيانات؟

### 2.6 تقني وتكامل

- **SEO:** عناوين ووصف ميتا جاهزة أم نعتمد نمطاً عاماً؟
- **تحليلات:** Google Analytics، Meta Pixel، غيرها؟ نعم/لا وأيها.
- **النماذج:** حفظ في DB فقط أم أيضاً إرسال إيميل إلى عنوان محدد؟ كابتشا (مثل reCAPTCHA)؟
- **أداء:** أي هدف محدد (مثلاً تحميل الرئيسية < 3 ثوانٍ على موبايل) أم "أداء جيد معياري"؟

### 2.7 الأولويات والمراحل

- **MVP (أول إطلاق):** أي الصفحات ضرورية من اليوم الأول؟
- **المرحلة الثانية:** ما الذي يأتي بعد ذلك (استوديو، أسئلة شائعة، مدونة، فلاتر متقدمة، إلخ).
- **الجدول الزمني:** تقدير (أسبوعان، شهر، شهران، أو بدون تاريخ ثابت).

### الحد الأدنى للبدء

1. قائمة الصفحات العامة (مثلاً: رئيسية، عن، اتصل، معدات، تسجيل دخول).
2. أقسام الصفحة الرئيسية (هيرو، قسم أو اثنين، CTA، فوتر).
3. الـ CTA الرئيسي: "اطلب عرض سعر" أم "سجّل دخول واحجز" أم الاثنين.
4. اللغة: عربي فقط، إنجليزي فقط، أو الاثنين مع افتراضي.
5. المحتوى: "استخدم placeholder" أم "سنوفّر النصوص والصور بتاريخ X".

## 3) المميزات من لوحة التحكم التي تؤثر على الموقع العام

| منطقة في اللوحة            | ما يظهر للجمهور                            | ملاحظات                                                                                                                                                           |
| -------------------------- | ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Inventory → Equipment**  | كتالوج معدات (قائمة + تفاصيل)              | **مصدر الكتالوج العام المُعتمد.** الـ API الحالي يتطلب auth → نحتاج قراءة عامة (catalog) للمعدات النشطة (isActive/featured). Equipment = المنطق التشغيلي والتوفر. |
| **Inventory → Products**   | واجهة تسويقية (ترجمات، SEO)                | Products واجهة تسويقية فقط؛ مصدر الحقيقة للعرض العام هو Equipment.                                                                                                |
| **Inventory → Categories** | فلاتر / تنقل في الكتالوج                   | نحتاج قائمة فئات عامة (مثلاً بدون حذف).                                                                                                                           |
| **Inventory → Brands**     | فلاتر / عرض في الكتالوج                    | نحتاج قائمة ماركات عامة.                                                                                                                                          |
| **Studios**                | قائمة استوديوهات + تفاصيل (+ توفر)         | الـ API حالياً mock؛ الـ DB فيه Studio (اسم، slug، وصف، media، isActive).                                                                                         |
| **Quotes**                 | "اطلب عرض سعر" من الموقع العام أو البورتال | الـ POST يتطلب auth → نحتاج endpoint عام لطلب عرض أو نموذج "اتصل بنا" يخلق Quote/Lead.                                                                            |
| **Coupons**                | تطبيق خصم عند الدفع (بورتال)               | يبقى خلف تسجيل دخول؛ لا يحتاج "عرض" إضافي للجمهور.                                                                                                                |
| **Bookings**               | إنشاء حجز من البورتال بعد عرض/دخول         | لا كتالوج حجز مباشر للجمهور؛ المسار: عام → عرض/دخول → بورتال.                                                                                                     |
| **Contracts / Invoices**   | عرض وتوقيع من البورتال                     | إدارة من اللوحة؛ الجمهور يحتاج فقط نقطة دخول واضحة ("سجّل دخول لمعرفة عقودك").                                                                                    |
| **Marketing**              | حملات تظهر على الموقع العام                | اختياري: بنر، عروض، أو صفحة هبوط لحملة.                                                                                                                           |

## 4) كيف نُظهر بيانات اللوحة في الموقع العام

### 4.1 المخزون والمعدات (Inventory)

| في اللوحة                                         | كيف نُظهره في الموقع العام                                                                                                                                                                                                  |
| ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **المعدات** (featured, isActive, صور، فئة، ماركة) | **الرئيسية:** قسم "الأكثر طلباً" أو "مميز" من المعدات (featured + isActive). **صفحة المعدات:** شبكة/قائمة للمعدات النشطة؛ فلاتر (فئة، ماركة، سعر). **صفحة المعدة:** تفاصيل، معرض صور، سعر، "أضف للحجز" أو "تحقق من التوفر". |
| **الفئات**                                        | قائمة/قائمة منسدلة في الهيدر أو الشريط الجانبي؛ روابط مثل `/equipment?category=...` أو `/equipment/cameras`.                                                                                                                |
| **الماركات**                                      | فلتر "الماركة" في الكتالوج؛ شعار في البطاقة أو صفحة التفاصيل.                                                                                                                                                               |
| **المنتجات** (واجهة تسويقية فقط)                  | مصدر الكتالوج العام = Equipment. إن وُجدت بيانات تسويقية من Product (ترجمات، SEO) يمكن دمجها لاحقاً؛ العرض والتوفر يُستمدان من Equipment.                                                                                   |
| **وسائط المعدات (Media)**                         | معرض صور في صفحة التفاصيل؛ صورة رئيسية في البطاقة.                                                                                                                                                                          |

### 4.2 الاستوديوهات

| في اللوحة                                                                | كيف نُظهره في الموقع العام                                                                                                                                                                                 |
| ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **الاستوديوهات** (اسم، وصف، صور، سعر/ساعة، سعة، isActive، إضافات، باقات) | **الرئيسية:** قسم "استوديوهاتنا" → رابط `/studios`. **صفحة الاستوديوهات:** قائمة استوديوهات نشطة؛ فلتر سعر/سعة. **صفحة الاستوديو:** تفاصيل، معرض، أسعار، باقات، إضافات، زر "احجز" → `/studio/{slug}/book`. |
| **باقات الاستوديو**                                                      | في صفحة الاستوديو: بطاقات "اختر باقة" (اسم، سعر افتراضي، مدة)؛ عند الاختيار تملأ التواريخ والإضافات.                                                                                                       |
| **إضافات الاستوديو**                                                     | في صفحة الحجز: قائمة إضافات اختيارية مع السعر؛ تُضاف للإجمالي.                                                                                                                                             |
| **أيام الإغلاق (StudioBlackoutDate)**                                    | في التقويم: التواريخ المحجوبة غير قابلة للاختيار أو تظهر "غير متاح".                                                                                                                                       |

### 4.3 التسعير والعروض

| في اللوحة                  | كيف نُظهره في الموقع العام                                                                                                                                                              |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **الكوبونات**              | في السلة أو صفحة الدفع: "لديك كود خصم؟" → إدخال → استدعاء API التحقق → تحديث الإجمالي وعرض "تم تطبيق الخصم".                                                                            |
| **التسعير الديناميكي**     | السعر المعروض في الكتالوج والتفاصيل والسلة = السعر المحسوب من القواعد (بدون كشف "القاعدة" للمستخدم).                                                                                    |
| **العروض/الباقات/Bundles** | **الرئيسية:** قسم "باقات وعروض". **صفحة الباقات:** قائمة باقات نشطة (اسم، محتويات، سعر "يبدأ من"، زر عرض/احجز). **في صفحة المعدة:** "اشترِ معاً" أو "باقة تحتوي هذه المعدة" مع التوفير. |

### 4.4 التسويق والإعدادات

| في اللوحة                                | كيف نُظهره في الموقع العام                                                                     |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------- |
| **الحملات**                              | بنر دوار أو شريط علوي في الرئيسية يعرض الحملة النشطة (صورة، عنوان، رابط). أو صفحة هبوط للحملة. |
| **الإعدادات العامة (Company, Branding)** | الهيدر والفوتر: شعار، ألوان، روابط تواصل وسوشال من الإعدادات.                                  |

### 4.5 الذكاء الاصطناعي والتوصيات

| في اللوحة            | كيف نُظهره في الموقع العام                                                                                                                                 |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **التوصيات (قواعد)** | في صفحة المعدة: قسم "يكمل معه" أو "يُشترى غالباً مع" مع خصم إن وُجد. في السلة: "أضف للتوفير" حسب المحتويات.                                                |
| **Kit Builder**      | صفحة "ابنِ كيتك" `/build-your-kit`: خطوات (استخدام → كاميرا → عدسة → إضاءة...) مع خيارات من المعدات النشطة وتوافق؛ في النهاية "أضف الكيت للحجز" وسعر مجمع. |

### 4.6 الإعدادات والقواعد

| في اللوحة                                             | كيف نُظهره في الموقع العام                                                                             |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **قواعد التوفر** (buffer, blackout, min/max duration) | في حجز الاستوديو: التقويم يحترم الـ buffer؛ مدة الحجز ضمن min/max. في المعدات: مدة الإيجار ضمن الحدود. |
| **Feature flags**                                     | إن كانت "Build Your Kit" مفعلة نُظهر الرابط والصفحة؛ إن أُوقفت نخفيها.                                 |

### 4.7 التوفر والموافقات

| في اللوحة            | كيف نُظهره في الموقع العام                                                                                           |
| -------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **التقويم/الحجوزات** | عند اختيار التواريخ نتحقق من التوفر ونعرض "متاح" / "غير متاح" أو عدد الوحدات.                                        |
| **الموافقات**        | إن كان الحجز "يحتاج موافقة": بعد الطلب/الدفع نعرض "جاري المراجعة" حتى يوافق من اللوحة ثم نحدّث الحالة ونبلّغ العميل. |

## 8) قرارات المنتج والـ UX

| المجال            | القرار                                | المعنى                                                                                                                                                                           |
| ----------------- | ------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **التوفر**        | Instant Book مع override من الخلفية   | الافتراضي: تأكيد فوري تلقائي. عند الحاجة يمكن من اللوحة جعل الحجز "يحتاج موافقة" حتى يتم الاعتماد يدوياً.                                                                        |
| **الصياغة**       | لغة وديّة، شروط صارمة                 | واجهة المستخدم: مصطلحات واضحة (مثل "تنازل عن الأضرار"، "الخصم المستقطع"). الشروط والأحكام صارمة ومتوافقة مع القانون السعودي.                                                     |
| **الدعم**         | دعم واعي للسياق                       | عند الضغط على "دعم/واتساب" يُرسل للوكيل السياق الحالي: محتويات السلة، الصفحة، رقم الحجز إن وُجد.                                                                                 |
| **تسجيل الدخول**  | موبايل أولاً OTP                      | الطريقة الأساسية: رقم جوال + OTP (SMS/واتساب). الإيميل/رابط سحري ثانوي أو اختياري.                                                                                               |
| **اللغات**        | عربي + إنجليزي + صيني                 | الموقع ثلاثي اللغات: العربية (RTL)، الإنجليزية (LTR)، الصينية (LTR). لغة افتراضية: عربي. تبديل اللغة في الهيدر.                                                                  |
| **ضيف vs حساب**   | تسجيل إجباري لكن **مؤجّل (Deferred)** | العميل يتصفح ويملأ السلة كضيف → يضغط "إتمام الحجز" → نطلب رقم الجوال + OTP → **يتم إنشاء الحساب والحجز في خطوة واحدة**. لا نطلب تسجيلاً قبل إتمام الحجز؛ الهدف رفع نسبة التحويل. |
| **مصدر الكتالوج** | **Inventory → Equipment**             | الكتالوج العام يعتمد على **Equipment** لأنها تحتوي المنطق التشغيلي (أرقام تسلسلية، توفر حقيقي). الـ **Products** واجهة تسويقية فقط؛ **Equipment** هي مصدر الحقيقة للعرض والتوفر. |

### 8.1 تدفق التسجيل المؤجّل (Deferred Registration)

```
[تصفح كضيف] → [إضافة للسلة] → [مراجعة السلة] → [إتمام الحجز]
                                                      ↓
                              [إدخال رقم الجوال] → [OTP] → [إنشاء حساب + الحجز معاً]
                                                      ↓
                              [دفع] → [تأكيد]
```

- لا نطلب تسجيل دخول أو إنشاء حساب قبل أن يصل المستخدم لخطوة "إتمام الحجز".
- عند "إتمام الحجز" نجمّع: رقم الجوال + OTP ثم ننشئ الحساب (إن لم يكن موجوداً) ونربط الحجز به في نفس الخطوة.
- بعدها يتابع الدفع كالمعتاد (ملخص، دفع، تأكيد).

## 9) متطلبات تجربة المستخدم (لتسهيل تجربة المستخدم)

### 9.1 التنقل والوضوح

- قائمة رئيسية 5–7 عناصر بلغة يومية (معدات، استوديوهات، باقات، كيف نعمل، اتصل بنا).
- مسارات واضحة من الرئيسية (تصفح المعدات، احجز استوديو، الباقات، ابنِ كيتك).
- خبز مكسور (Breadcrumbs) في التصنيفات وصفحات التفاصيل.

### 9.2 الكتالوج والبحث

- فلاتر بسيطة (نوع، ماركة، سعر، "متاح للتواريخ المختارة").
- ترتيب (الأحدث، السعر، الأكثر طلباً).
- بطاقة منتج: صورة، اسم، سعر/يوم، إشارة توفر، زر "عرض".
- بحث مع اقتراحات أو تصحيح.

### 9.3 التواريخ والتوفر

- انتقاء تاريخ واضح (موبايل أولاً): تقويم، اليوم واضح، تنقل أشهر، تحديد فترة (من–إلى) بلون.
- تنسيق تاريخ محلي (عربي/سعودي).
- بعد اختيار التواريخ عرض "متاح" / "غير متاح" / "عدد محدود" فوراً.
- عند عدم التوفر: رسالة واضحة + اقتراح تواريخ بديلة أو "تواصل بنا".

### 9.4 السعر والشفافية

- السعر الإجمالي ظاهر من البداية (في القائمة والتفاصيل).
- قبل الدفع: تفصيل (إيجار، خصم، تنازل/خصم مستقطع، ضريبة، إجمالي).
- عدم ظهور رسوم غير متوقعة في الخطوة الأخيرة.
- إن وُجد "قفل سعر" بوقت: إظهاره بوضوح (مثلاً "السعر ثابت لمدة 10 دقائق").

### 9.5 السلة والدفع (تقليل الاحتكاك)

- ملخص الطلب ثابت أثناء الدفع.
- أقل حقول ممكنة؛ تحقق فوري من الحقول.
- حقل كوبون منفصل ("لديك كود؟") مع تحديث الإجمالي فوراً.
- زر دفع واضح ("ادفع X ريال وتأكيد الحجز").
- إن كان الدفع متعدد الخطوات: مؤشر تقدم (1–2–3).

### 9.6 الثقة والأمان

- إشارات أمان بسيطة عند الدفع (قفل، "دفع آمن") دون تكديس شارات.
- شعارات وسائل الدفع المعتمدة.
- روابط الشروط وسياسة الإلغاء قريبة من زر التأكيد وفي الفوتر.

### 9.7 الدعم والسياق

- زر دعم/واتساب يرسل السياق (السلة، الصفحة، رقم الحجز).
- وضوح توفر الدعم (مثلاً "متاح من X إلى Y" أو "رد خلال ساعات").
- أسئلة شائعة قصيرة (رئيسية أو "كيف نعمل") مع روابط من صفحات الحجز.

### 9.8 موبايل و RTL

- تصميم RTL كامل (قوائم يمين، أزرار رئيسية يمين).
- أزرار وأهداف لمس كبيرة (≈44px) ومسافات كافية.
- تقويم وانتقاء تاريخ مريح على الموبايل.
- تسجيل دخول موبايل أولاً (رقم جوال + OTP).

### 9.9 الأداء

- تحميل سريع؛ صور محسّنة (مثلاً Next/Image).
- هيكل تحميل (Skeleton) أثناء جلب البيانات.
- حالات فارغة وخطأ مع رسالة وزر "إعادة المحاولة" أو اقتراح تغيير الفلتر.

### 9.10 بعد الحجز

- صفحة تأكيد واضحة ("تم تأكيد حجزك"، ملخص، تواريخ، استلام/إرجاع، رقم حجز).
- تحميل PDF أو مشاركة/إضافة للتقويم.
- رابط واتساب للدعم مع رقم الحجز.
- (اختياري) تذكير بالسلة المهجورة بعد 1–2 ساعة مع ملخص ورابط إكمال، وربما خصم محدود.

## 10) المعلومات المتبقية المطلوبة

لتكميل التخطيط والتنفيذ بدقة، ما زلنا نحتاج توضيحاً في التالي:

### 10.1 النطاق والإطلاق

- **صفحات الـ MVP:** أي الصفحات ضرورية من اليوم الأول؟
- **ضيف vs حساب:** ✅ **مُحدد:** تسجيل إجباري مؤجّل – التصفح والسلة كضيف، وعند "إتمام الحجز" نطلب رقم جوال + OTP ثم ننشئ الحساب والحجز معاً (انظر القسم 8).
- **الاستوديوهات عند الإطلاق:** نعتمد بيانات حقيقية من اللوحة من اليوم الأول أم نطلق بدون استوديو (معدات فقط) ثم نضيف الاستوديو لاحقاً؟

### 10.2 المحتوى والنصوص

- نصوص الصفحة الرئيسية (عنوان، وصف، أقسام): جاهزة أم placeholder؟
- الصفحات الثابتة: عن، اتصل بنا، أسئلة شائعة، سياسة خصوصية، شروط وأحكام – والنص القانوني (سعودي) جاهز أم نضع هيكل فقط؟
- اللغة الافتراضية: ✅ **مُحدد:** عربي. اللغات المدعومة: **عربي + إنجليزي + صيني** (ثلاثي اللغات).

### 10.3 الهوية

- الشعار (نسخة فاتح/داكن إن وُجد).
- الألوان (هيكس) أو "نحافظ على الحالي" أو "اقترح palette".
- الخطوط: نكمل Cairo + Inter أم أخرى؟

### 10.4 تقني وقرارات منتج

- **مصدر الكتالوج العام:** ✅ **مُحدد:** **Inventory → Equipment** (isActive, featured). Equipment = المنطق التشغيلي والتوفر؛ Products = واجهة تسويقية فقط (انظر القسم 8).
- **APIs للصفحة العامة:** هل نجهّز endpoints عامة (بدون تسجيل) لقراءة المعدات/الاستوديوهات/الفئات/العلامات؟
- **OTP وتسجيل الدخول:** مزود محدد (Twilio، مزود سعودي، واتساب) أم نصمم الواجهة فقط ونترك التكامل لاحقاً؟

### 10.5 الوسائط

- صور المعدات/الاستوديوهات: من اللوحة فقط أم مجموعة placeholder حتى تكتمل البيانات؟
- صور رئيسية (هيرو، أقسام): stock أم صور محددة منكم؟

### 10.6 الدعم

- ربط زر الدعم الواعي للسياق: رقم واتساب ثابت أم حساب واتساب Business (رابط + params)؟
- نص أوقات الاستجابة للمستخدم (مثل "رد خلال 24 ساعة").

### 10.7 التواريخ والأولويات

- موعد مستهدف تقريبي للإطلاق (أسبوعان، شهر، شهران).
- أهم 3 ميزات بعد الإطلاق (باقات، ابنِ كيتك، توصيل، عروض، إلخ).
